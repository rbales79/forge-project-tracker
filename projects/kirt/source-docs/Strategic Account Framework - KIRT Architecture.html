Project Overview

Foundation — Data Layer

Data platform, not application. Ingests from OneDrive, SharePoint, file drops, and Forge app outputs. Parses, embeds, stores raw source as canonical record, and serves all consumers via 5 stable API endpoints. No business logic — consuming apps own all workflow and decision logic.

🏔️ Lakehouse

Apache Iceberg on Parquet / Azure Blob

Confirmed

Source of truth. Raw content preserved — chunks are derived views, never canonical. Apache Polaris catalog (REST, Postgres-backed). DuckDB query engine for transforms.

🐍 Backend & Pipeline

Django REST Framework + Celery

Confirmed

DRF ViewSets for 5 stable API endpoints (ingest, query, retrieve, list, delete). Celery task chains for async ingestion pipeline (4 worker queues). Django admin for operational monitoring. Tenant-scoped from day one.

📄 Parsing & Embedding

Docling + LlamaIndex + Production Embeddings

Confirmed

Docling for complex layouts (PDF, PPTX — table extraction, reading order). LlamaIndex loaders for 7 additional formats. OpenAI or Cohere embeddings (swappable provider). Content-type routing to chunk strategy (semantic, slide, transcript, structured).

🔗 Connectors

Microsoft Graph API + File Drop

Confirmed

OneDrive + SharePoint delta sync via MSAL + httpx. File Drop for CSV exports and uploaded documents. 9 file formats supported. Entity extraction from folder paths and SP metadata.

KIRT — Intelligence Layer

Where the business intelligence happens. All classification, scoring, template routing, deliverable generation, and user-facing features. Queries Foundation's data via API and Weaviate.

⚛️ Frontend

React + Vite + Tailwind + shadcn

Confirmed

🧠 Semantic Search & RAG

Weaviate (via Foundation)

Confirmed

Confirmed

Adoption Strategy

Success requires more than a well-built system—it requires users who actively engage with it. KIRT must answer the fundamental question: What makes people want to use it, and how do we get them to upload their notes, transcripts, and planning documents?

The Value Exchange Framework

Every interaction must provide immediate, visible value. The feedback loop between upload and output must be tight, obvious, and compelling.

💡 Immediate Value Exchange

Every upload produces something useful within minutes:

  • Upload meeting notes → get an engagement prep brief
  • Upload transcript → get cross-sell recommendations
  • Input company name → get client deliverable documentation

Design principle: The user's first experience should be "this saved me hours."

⚡ Reduced Effort, Not Added Process

KIRT replaces manual work, never adds steps. The interaction model is:

  • Account exec walks into meeting, prints client deliverable documentation (not: fill out form)
  • Upload existing meeting notes (not: retake the meeting)
  • Automatic transcript capture (not: manually transcribe)

Success metric: Time-to-usefulness < 2 minutes from first login.

🏆 Social Proof & FOMO

Early adopters visibly outperform. Competitive pressure drives adoption:

  • First users generate better deliverables 80%+ faster
  • Cross-sell recommendations surface deals others miss
  • Account briefs are richer and more current

Design principle: Make early wins visible—Slack messages, team calls, leaderboards of usage.

📈 Low Floor, High Ceiling

Works with minimal input; gets dramatically better with more data:

  • Day 1: "Company name" → reasonable brief
  • Week 1: Add 2-3 transcripts → personalized recommendations
  • Month 1: Full integration → predictive alerts, pattern matching

Design principle: No minimum viable effort—every bit of data helps.

Familiar Patterns & Mental Models

KIRT leverages interaction models people already know and trust:

🔍 Search-Box Simplicity

Discovery and intelligence access through a single search interface, like Google or Salesforce. No forms, no navigation trees.

📦 Amazon-Style Recommendations

"Customers like yours also bought..." pattern applied to services. Invisible pattern recognition that feels helpful, not intrusive.

⏱️ One-Click Generation

No multi-step wizards. Account brief, engagement prep, client deliverable documentation—all available from a single click.

🔄 Feedback Loops

User feedback improves the system in real-time. Edit a section? System learns your preferences for future generations.

Network Effects & Compounding Value

Every upload makes KIRT smarter for that user AND for their teammates:

  • Individual value: Dwayne uploads a transcript where he suggested a PCI assessment as part of a storage sale → next time Roy preps for a meeting to sell storage in a regulated environment, the system recommends PCI → Roy gets credit for the cross-sell
  • Team value: That transcript improves patterns for every rep selling storage solutions in regulated environments → everyone benefits
  • Organizational value: Patterns aggregate across regions, practices, and customer types → best practices surface automatically

Success bar: "So wildly successful that if it disappears one day, everybody's like, 'What happened?'"

Reclaim Time for Revenue-Producing Work

If the platform saves each GTM team member at least 10 minutes per day by removing friction from low-value tasks — searching for information, recreating materials, navigating internal systems — the cumulative impact is significant.

This is not about people working less. It is about redirecting time toward work that actually drives revenue: advancing deals, engaging customers, building solutions, and closing business.

Rollout Phasing

Phase 2: Extended Leadership (80-120 people)

Phase 3: Broad Rollout (300-500 people)

General availability to all account executives, architects, pre-sales. Word-of-mouth from high-performing peers. Social proof is the adoption driver.

Risks & Gaps

Key risks and known gaps that could impact timeline, scope, or success criteria.

HIGH Data Normalization Crisis

Problem: Customer records are duplicated and inconsistent across SharePoint, Salesforce, and CRM systems

Impact: Blocks Salesforce API integration, makes report exports unreliable, degrades brief quality

Mitigation: Phase 1 uses manual export + data cleansing scripts; consider master data governance post-MVP

HIGH No Cost Model

Problem: Board will expect specific financial ROI and cost justification

Impact: Cannot defend 50-70% reduction in effort claim without backing math

Mitigation: Build cost model in parallel; interview 5-10 sales leaders on time/cost savings opportunity

HIGH Open Question: Cost of Sale Definition

Question: What's included in "cost of sale"? How is it measured? How does it factor in AEs, Solution Specialists, and Practice Support for GTM from non-billable resources?

Impact: Cannot accurately calculate ROI without clarity on cost inputs and allocation methodology

Next Step: Clarify with finance and sales leadership during cost model development

MEDIUM Legal/IT Approval Path

Problem: Data ingestion, AI use, and vendor integration require compliance reviews

Impact: Could delay Phase 1 or require architectural changes

Mitigation: Engage legal/IT immediately; use pre-approved Azure services

MEDIUM No Intermediate KPIs

Problem: Difficult to measure progress between now and the Rusk pitch meetings

Impact: Meetings lack objective success criteria and demonstrated wins

Mitigation: Define sprint-level metrics; weekly dashboard of build progress

MEDIUM Synthetic Data Strategy

Problem: Real customer data is sensitive; synthetic data generation for demos not yet planned

Impact: Cannot demo system safely with real data, or demo will lack realism

Mitigation: Build synthetic data generator; validate with small real dataset

LOW Competitive Landscape Not Validated

Problem: Market research on competing AI account intelligence tools is incomplete

Impact: Board may compare KIRT to products with different architecture/scope

Mitigation: Quick competitive scan in Week 1; document differentiation

LOW Product Naming Not Settled

Problem: Naming inconsistency: KIRT vs Forge Foundation (legacy naming resolved)

Impact: Communications confusion, brand dilution

Mitigation: Decide final name by Week 2; use consistently in all materials
